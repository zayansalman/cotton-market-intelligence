# Regulatory Compliance Assessment

## Applicable Regulations

CMI generates AI-powered commodity procurement recommendations. Depending on jurisdiction and use case, the following regulations may apply:

### EU AI Act (fully applicable August 2026)

**Classification:** CMI is likely a **limited-risk AI system** under the EU AI Act. It interacts directly with users to provide commodity advisory output, triggering transparency obligations.

**Requirements:**
- **Transparency:** Users must be clearly informed they are interacting with an AI system
- **Explainability:** AI outputs must be traceable to input data and reasoning
- **Human oversight:** A human must be able to override or disregard AI recommendations
- **Data governance:** Training data and model decisions must be documented

**CMI compliance status:**
- Transparency: Model name and provider shown in UI (Qwen 2.5 7B via HF Pro)
- Explainability: Methodology breakdown shows signal-by-signal analysis
- Human oversight: All recommendations are advisory — user makes final decision
- Data governance: All data sources documented in wiki/V3-Data-Dictionary.md

### MiFID II / MiFID III (EU)

**Applicability:** CMI provides commodity procurement recommendations, not investment advice on financial instruments. However, if cotton futures positions are taken based on CMI output, MiFID II conduct of business rules may apply.

**Key requirements:**
- Best interest of client obligation
- Suitability assessment for personalized recommendations
- Record-keeping of recommendations provided
- Fair, clear, and not misleading communications

**CMI compliance status:**
- Disclaimers: Added to UI stating output is not investment advice
- Suitability: Not applicable (procurement tool, not personal investment advice)
- Record-keeping: Audit metadata included in API responses (timestamp, model, inputs)
- Fair communication: Confidence levels and risks shown alongside recommendations

### FCA Consumer Duty (UK)

**Requirements:**
- Act in good faith toward retail customers
- Avoid causing foreseeable harm
- Enable customers to pursue financial objectives
- Empirically demonstrate good outcomes

**CMI compliance status:**
- Good faith: Transparent methodology with reasoning shown
- Foreseeable harm: Risk factors explicitly stated for every recommendation
- Financial objectives: Procurement cost optimization framed correctly
- Senior Manager accountability: User is the decision maker, CMI is advisory

### BaFin (Germany)

**Requirements:**
- Algorithmic trading systems must maintain audit trails
- Model risk management frameworks required
- Outsourcing to AI providers must be documented

**CMI compliance status:**
- Audit trail: API responses include timestamp, model ID, input data hash
- Model risk: Walk-forward backtesting and accuracy scorecard documented
- Provider documentation: HF Pro, Qwen model, featherless-ai provider documented

## Required Disclaimers

The following disclaimers are displayed in the CMI UI:

1. **AI Advisory Disclaimer:** "This tool provides AI-generated market analysis and procurement recommendations. It is not investment advice. All recommendations should be validated by qualified professionals before execution."

2. **Model Limitation Disclaimer:** "Predictions are generated by AI models (Qwen 2.5 7B) analyzing market data and news. Models can be wrong. Past performance does not guarantee future results."

3. **No Guarantee Disclaimer:** "Cotton Market Intelligence provides analytical support for procurement decisions. It does not guarantee any specific price outcome or return."

4. **Data Source Disclaimer:** "Market data sourced from Yahoo Finance (delayed). News from public RSS feeds. Sentiment via Hugging Face NLP models. Cross-market signals are directional indicators, not precise forecasts."

## Audit Trail Requirements

Every API response includes:
- `generated_at`: ISO timestamp of generation
- `version`: API version number
- `model.id`: Which model produced the output
- `model.name`: Human-readable model name
- `methodology`: Signal-by-signal analytical breakdown (when available)
- `confidence`: Stated confidence level
- `risk`: Explicit risk to the forecast
- `cross_market_signals`: Raw data the model saw

## Compliance Gaps and Remediation

| Gap | Severity | Remediation |
|-----|----------|-------------|
| No persistent audit log (responses not stored) | Medium | Add server-side logging of all strategy/prediction requests |
| No user authentication (anyone can access) | Low | Add auth when tool is used for commercial procurement |
| No suitability assessment | Low | Not required for B2B procurement tool (not retail investment advice) |
| No formal model validation report | Medium | Generate from walk-forward backtest results |
| GDPR: no data processing agreement with HF | Medium | Review HF Pro DPA terms |
